# Daily Wrap-Up

## When to use

- End of each workday (automatic loop: weekdays 6pm)
- After saving a file in `notes/` (automatic hook)
- Manually: run `scripts/daily-wrap-up.sh` or ask Claude to run the skill

## Steps

1. **Find today's notes** — `notes/YYYY-MM-DD.md` (today's date in EAT)
2. **Read** all bullet points and sections
3. **Extract** three buckets:
   - **Done** — completed today
   - **Doing** — in progress, not finished
   - **Next** — tomorrow or follow-up actions
4. **Write summary** to `log/YYYY-MM-DD-wrapup.md`
5. **Keep it short** — 3–5 bullets per section max

## Output format

```markdown
# Daily Wrap-Up — YYYY-MM-DD

## Done
- item

## Doing
- item

## Next
- item

---
*Generated by daily-wrap-up (trigger: manual|loop|hook)*
```

## Example

**Input** (`notes/2026-07-09.md`):
- Completed Memory Tune-Up assessment
- Working on Module 6 Daily Wrap-Up

**Output** (`log/2026-07-09-wrapup.md`):
```markdown
# Daily Wrap-Up — 2026-07-09

## Done
- Memory Tune-Up assessment (all 5 tasks)
- Pushed memory-tuneup-kitili-mbula repo

## Doing
- Module 6 skill + loop + hook setup

## Next
- Upload Google Doc reflections
- Add taleemabad-university as collaborator
```
